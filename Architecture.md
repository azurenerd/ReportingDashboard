# Architecture

## Overview & Goals

This is a single-page executive reporting dashboard that renders a pixel-perfect release roadmap visualization at 1920×1080 resolution, designed for screenshot capture into PowerPoint decks. The application is built with C# .NET 8 Blazor Server, runs entirely locally with zero cloud dependencies, and reads all data from a single `data.json` file on disk.

**Primary Goals:**
1. Pixel-perfect visual match to `OriginalDesignConcept.html` at 1920×1080
2. Data-driven rendering — edit `data.json`, refresh browser, see updated dashboard
3. Zero-ceremony operation — `dotnet run` and open a browser, nothing else
4. No external dependencies — no databases, no cloud services, no NPM packages, no charting libraries

**Non-Goals (Explicitly Out of Scope):**
- Authentication, authorization, or user management
- REST/GraphQL API endpoints
- Database or ORM integration
- Mobile/responsive layout
- Real-time updates or SignalR push
- Multi-page routing
- Docker, CI/CD, or cloud deployment
- Dark mode or theme switching

## System Components

### 1. Data Layer

#### 1.1 `data.json` (Configuration File)
- **Responsibility:** Single source of truth for all dashboard content — title, subtitle, milestones, timeline events, heatmap categories, month columns, and metadata links.
- **Location:** Project root, next to `.csproj`. Copied to output directory on build.
- **Format:** UTF-8 JSON, hand-edited or generated by an external process.
- **Interfaces:** Read by `DashboardDataService` via `System.IO.File.ReadAllTextAsync`.
- **Dependencies:** None (static file).

#### 1.2 `DashboardData.cs` (Data Models)
- **Responsibility:** Strongly-typed C# record types that mirror the `data.json` schema. Provide compile-time safety and IDE support.
- **Location:** `Models/DashboardData.cs`
- **Dependencies:** `System.Text.Json.Serialization` for JSON property mapping.

**Record definitions:**

```csharp
namespace ReportingDashboard.Models;

public record DashboardData
{
    public required string Title { get; init; }
    public required string Subtitle { get; init; }
    public required string BacklogUrl { get; init; }
    public required string CurrentDate { get; init; }
    public required List<Milestone> Milestones { get; init; }
    public required List<string> Months { get; init; }
    public required string CurrentMonth { get; init; }
    public required List<StatusCategory> Categories { get; init; }
    public required TimelineConfig Timeline { get; init; }
}

public record Milestone
{
    public required string Id { get; init; }
    public required string Label { get; init; }
    public required string Color { get; init; }
    public required List<MilestoneEvent> Events { get; init; }
}

public record MilestoneEvent
{
    public required string Date { get; init; }
    public required string Type { get; init; }  // "checkpoint" | "poc" | "production"
    public required string Label { get; init; }
}

public record StatusCategory
{
    public required string Name { get; init; }
    public required string CssClass { get; init; }
    public required Dictionary<string, List<string>> Items { get; init; }

public record TimelineConfig
{
    public required string StartDate { get; init; }
    public required string EndDate { get; init; }
    public required int SvgWidth { get; init; }    // default: 1560
    public required int SvgHeight { get; init; }   // default: 185

```

#### 1.3 `DashboardDataService.cs` (Data Access Service)
- **Responsibility:** Reads `data.json` from disk, deserializes into `DashboardData`, provides computed layout values (SVG coordinates, grid dimensions).
- **Location:** `Services/DashboardDataService.cs`
- **Interfaces:** Registered in DI as a scoped service. Consumed by `Dashboard.razor` page component.
- **Dependencies:** `System.IO`, `System.Text.Json`, `Models/DashboardData.cs`.

**Key methods:**

```csharp
namespace ReportingDashboard.Services;

public class DashboardDataService
{
    private readonly string _dataFilePath;

    public DashboardDataService(IWebHostEnvironment env)
    {
        _dataFilePath = Path.Combine(env.ContentRootPath, "data.json");
    }

    public async Task<DashboardData> LoadAsync()
    {
        var json = await File.ReadAllTextAsync(_dataFilePath);
        return JsonSerializer.Deserialize<DashboardData>(json, new JsonSerializerOptions
        {
            PropertyNameCaseInsensitive = true
        }) ?? throw new InvalidOperationException("Failed to deserialize data.json");
    }

    public double CalculateXPosition(string dateStr, string startDate, string endDate, int svgWidth)
    {
        var date = DateOnly.Parse(dateStr);
        var start = DateOnly.Parse(startDate);
        var end = DateOnly.Parse(endDate);
        var totalDays = end.DayNumber - start.DayNumber;
        var elapsed = date.DayNumber - start.DayNumber;
        return totalDays > 0 ? (double)elapsed / totalDays * svgWidth : 0;
    }

    public double CalculateNowMarkerX(string startDate, string endDate, int svgWidth)
    {
        return CalculateXPosition(
            DateOnly.FromDateTime(DateTime.Now).ToString("yyyy-MM-dd"),
            startDate, endDate, svgWidth);
    }

    public string BuildGridTemplateColumns(int monthCount)
    {
        return $"160px repeat({monthCount},1fr)";
    }
}
```

### 2. Presentation Layer (Blazor Components)

#### 2.1 `App.razor` (Application Shell)
- **Responsibility:** Root component. Sets up HTML `<head>` with CSS reference (no viewport meta tag), renders `<body>` with the `Dashboard` page. Configures static SSR render mode.
- **Dependencies:** `dashboard.css`, `Routes.razor`.

#### 2.2 `Dashboard.razor` (Page Orchestrator)
- **Responsibility:** Top-level page component. Injects `DashboardDataService`, calls `LoadAsync()` in `OnInitializedAsync`, distributes data to child components via `[Parameter]` attributes.
- **Route:** `@page "/"` (sole route in the application).
- **Data:** Holds `DashboardData` instance. Passes slices to children.
- **Dependencies:** `DashboardDataService`, all child layout components.

#### 2.3 `Header.razor` (Header Bar)
- **Responsibility:** Renders the top bar containing the dashboard title with ADO backlog hyperlink, subtitle line, and the legend icons (PoC Milestone diamond, Production Release diamond, Checkpoint circle, Now marker line).
- **Parameters:**
  - `string Title`
  - `string Subtitle`
  - `string BacklogUrl`
- **CSS class:** `.hdr`

#### 2.4 `TimelineSection.razor` (SVG Gantt Chart)
- **Responsibility:** Renders the milestone swim-lane labels (left sidebar) and the SVG timeline area with month grid lines, milestone horizontal lines, event markers (circles, diamonds), date labels, and the red dashed "NOW" marker.
- **Parameters:**
  - `List<Milestone> Milestones`
  - `TimelineConfig Timeline`
  - `string CurrentDate`
  - `List<string> Months`
- **Computed values:** SVG X-coordinates for each event, Y-coordinates per swim lane (evenly distributed), NOW marker X-position.
- **CSS class:** `.tl-area`, `.tl-svg-box`

#### 2.5 `HeatmapGrid.razor` (Status Matrix Container)
- **Responsibility:** Renders the heatmap section title ("Status by Category"), the CSS Grid container with corner cell, column headers (months, with current month highlighted), and delegates each row to `HeatmapRow.razor`.
- **Parameters:**
  - `List<StatusCategory> Categories`
  - `List<string> Months`
  - `string CurrentMonth`
- **Dynamic CSS:** `grid-template-columns` is computed from month count. `grid-template-rows` is computed from category count.
- **CSS class:** `.hm-wrap`, `.hm-grid`

#### 2.6 `HeatmapRow.razor` (Status Row)
- **Responsibility:** Renders a single category row — the row header cell (e.g., "SHIPPED") and one `HeatmapCell` per month.
- **Parameters:**
  - `StatusCategory Category`
  - `List<string> Months`
  - `string CurrentMonth`
- **CSS classes:** `.hm-row-hdr`, row-specific classes (`.ship-hdr`, `.prog-hdr`, `.carry-hdr`, `.block-hdr`)

#### 2.7 `HeatmapCell.razor` (Individual Grid Cell)
- **Responsibility:** Renders a single data cell containing a list of bullet-point items with colored dot indicators.
- **Parameters:**
  - `List<string> Items`
  - `string CssClass` (e.g., `"ship-cell"`)
  - `bool IsCurrentMonth`
- **CSS classes:** `.hm-cell`, category-specific classes, `.apr` modifier for current month highlight.

### 3. Static Assets

#### 3.1 `dashboard.css` (Global Stylesheet)
- **Location:** `wwwroot/css/dashboard.css`
- **Responsibility:** Contains all CSS ported directly from `OriginalDesignConcept.html`. This is a 1:1 copy with one modification: `grid-template-columns` is removed from `.hm-grid` (applied dynamically via inline style from data).
- **Key design tokens preserved:**
  - Body: `width:1920px; height:1080px; overflow:hidden`
  - Font: `'Segoe UI', Arial, sans-serif`
  - Colors: Green (#34A853, #1B7A28, #E8F5E9, #F0FBF0, #D8F2DA), Blue (#0078D4, #1565C0, #E3F2FD, #EEF4FE, #DAE8FB), Amber (#F4B400, #B45309, #FFF8E1, #FFFDE7, #FFF0B0), Red (#EA4335, #991B1B, #FEF2F2, #FFF5F5, #FFE4E4)

| **Heatmap row: In Progress** (blue) | `HeatmapRow.razor` | Same grid row pattern. `.prog-hdr` + `.prog-cell` classes. | `Category` where `CssClass == "prog"`. | None |
| **Heatmap row: Carryover** (amber) | `HeatmapRow.razor` | Same grid row pattern. `.carry-hdr` + `.carry-cell` classes. | `Category` where `CssClass == "carry"`. | None |
| **Heatmap row: Blockers** (red) | `HeatmapRow.razor` | Same grid row pattern. `.block-hdr` + `.block-cell` classes. | `Category` where `CssClass == "block"`. | None |

Each item is a `<div class="it">` with `::before` pseudo-element for colored bullet. | `Items` string list rendered as `@foreach`. `IsCurrentMonth` adds `.apr` CSS modifier for highlighted background. | None |

### Dynamic CSS Generation

The heatmap grid dimensions are data-driven:

```csharp
// In HeatmapGrid.razor
@{
    var colTemplate = $"160px repeat({Months.Count}, 1fr)";
    var rowTemplate = $"36px repeat({Categories.Count}, 1fr)";
}

<div class="hm-grid" style="grid-template-columns: @colTemplate; grid-template-rows: @rowTemplate;">
```

### SVG Rendering Strategy

All SVG elements are rendered in `TimelineSection.razor` using Blazor markup with C# expressions:

```
SVG Element Type        Design Element              Razor Binding
──────────────          ──────────                  ─────────────
<line> (vertical)       Month grid dividers         X from date-to-pixel math
<line> (horizontal)     Milestone swim lanes        Y from lane index × spacing
<circle> (open)         Checkpoint markers          CX from event date, stroke from milestone color
<circle> (filled)       Small progress dots         CX from event date, fill #999
<polygon>               Diamond milestones          Points computed from center X,Y ± 11px
<text>                  Month labels, event labels  X,Y positioned relative to markers
<line> (dashed red)     NOW marker                  X from DateTime.Now calculation
<filter id="sh">        Drop shadow on diamonds     Static SVG <defs> block
```

## Component Interactions

### Data Flow (Top-Down, Unidirectional)

```
Program.cs
  │  Registers DashboardDataService in DI container
  │
  ▼
App.razor
  │  Renders <Routes> → resolves "/" to Dashboard.razor
  │
  ▼
Dashboard.razor [@page "/"]
  │  OnInitializedAsync:
  │    data = await DashboardDataService.LoadAsync()
  │    nowX = DashboardDataService.CalculateNowMarkerX(...)
  │    gridCols = DashboardDataService.BuildGridTemplateColumns(...)
  │
  ├──→ Header.razor
  │      [Parameter] Title, Subtitle, BacklogUrl
  │
  ├──→ TimelineSection.razor
  │      [Parameter] Milestones, Timeline, CurrentDate, Months
  │      Internally computes: X positions per event, Y positions per lane, NOW marker X
  │
  └──→ HeatmapGrid.razor
         [Parameter] Categories, Months, CurrentMonth
         │
         ├──→ (column headers rendered inline)
         │
         └──→ HeatmapRow.razor (one per category)
                [Parameter] Category, Months, CurrentMonth
                │
                └──→ HeatmapCell.razor (one per month per category)
                       [Parameter] Items, CssClass, IsCurrentMonth
```

### Communication Pattern

- **No callbacks, no events, no two-way binding.** This is a read-only dashboard.
- All data flows top-down via `[Parameter]` attributes.
- No component state mutations after initial render.
- No SignalR interactivity needed — static SSR render mode is sufficient.

### Service Lifetime

| Service | DI Lifetime | Justification |
|---------|------------|---------------|
| `DashboardDataService` | **Scoped** | Reads file per request, allowing `data.json` edits between refreshes |

### Storage

- **No database.** All data lives in `data.json` on the local filesystem.
- File is read on every page load (scoped service lifetime), enabling live edits without restart.
- No ORM, no migrations, no connection strings.

## API Contracts

**There are no API endpoints.** This application has no REST, GraphQL, or gRPC interfaces.

The sole "contract" is the `data.json` file schema, which serves as the interface between the data author (human or external tool) and the dashboard renderer.

### `data.json` Schema Contract

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["title", "subtitle", "backlogUrl", "currentDate", "milestones", "months", "currentMonth", "categories", "timeline"],
  "properties": {
    "title": { "type": "string" },
    "subtitle": { "type": "string" },
    "backlogUrl": { "type": "string", "format": "uri" },
    "currentDate": { "type": "string", "format": "date" },
    "months": { "type": "array", "items": { "type": "string" }, "minItems": 1 },
    "currentMonth": { "type": "string" },
    "milestones": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["id", "label", "color", "events"],
        "properties": {
          "id": { "type": "string" },
          "label": { "type": "string" },
          "color": { "type": "string", "pattern": "^#[0-9A-Fa-f]{6}$" },
          "events": {
            "type": "array",
            "items": {
              "type": "object",
              "required": ["date", "type", "label"],
              "properties": {
                "date": { "type": "string", "format": "date" },
                "type": { "type": "string", "enum": ["checkpoint", "poc", "production"] },
                "label": { "type": "string" }
              }
            }
          }
        }
      }
    },
    "categories": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["name", "cssClass", "items"],
        "properties": {
          "name": { "type": "string" },
          "cssClass": { "type": "string", "enum": ["ship", "prog", "carry", "block"] },
          "items": {
            "type": "object",
            "additionalProperties": { "type": "array", "items": { "type": "string" } }
          }
        }
      }
    },
    "timeline": {
      "type": "object",
      "required": ["startDate", "endDate", "svgWidth", "svgHeight"],
      "properties": {
        "startDate": { "type": "string", "format": "date" },
        "endDate": { "type": "string", "format": "date" },
        "svgWidth": { "type": "integer", "default": 1560 },
        "svgHeight": { "type": "integer", "default": 185 }
      }
    }
  }
}
```

### Error Handling

| Failure Mode | Behavior |
|---|---|
| `data.json` missing | Application throws `FileNotFoundException` on startup. Console error message with expected path. |
| `data.json` malformed JSON | `JsonException` with line/position info. Dashboard shows error page with message. |
| Missing required field | `JsonException` from `required` record properties. Descriptive error at deserialization. |
| Invalid date format | `FormatException` from `DateOnly.Parse`. Caught in `TimelineSection` with fallback to X=0. |
| Empty milestones/categories | Dashboard renders with empty sections. No crash — `@foreach` over empty list produces no output. |

- Single file: `data.json` (~2-10 KB depending on item count).
- No log files, no temp files, no cache directories.

### CI/CD

- **Not required.** This is a local development tool.
- If desired later: `dotnet build` and `dotnet test` are the only pipeline steps.

## Technology Stack Decisions

| Decision | Choice | Justification |
|---|---|---|
| **UI Framework** | Blazor Server (.NET 8) | Mandatory per requirements. Static SSR mode ideal for read-only dashboard. |
| **Render Mode** | Static SSR | No interactivity needed. Renders once on server, sends HTML to browser. Minimal resource usage. |
| **CSS Strategy** | Global `dashboard.css` | The design CSS is a single cohesive unit. Scoped CSS with `::deep` adds friction for no benefit. Global file enables 1:1 port from reference HTML. |
| **Timeline Rendering** | Inline SVG in Razor | Full pixel control. The design SVG uses basic primitives (`<line>`, `<circle>`, `<polygon>`, `<text>`) that Blazor renders natively. No charting library matches the design's exact positioning. |
| **Heatmap Layout** | CSS Grid | Direct match to the design's `grid-template-columns: 160px repeat(N,1fr)`. Column count is data-driven. |
| **Data Source** | `data.json` file | No database needed for monthly-cadence reporting data. File is human-editable, versionable, and requires zero infrastructure. |
| **JSON Library** | `System.Text.Json` | Built into .NET 8. No NuGet package needed. Handles all deserialization needs. |
| **Data Models** | C# 12 records with `required` | Immutable by default, concise syntax, built-in JSON deserialization validation via `required` keyword. |
| **Charting Library** | None | The timeline is ~20 SVG elements. A charting library would add dependency weight and reduce pixel-level control. |
| **CSS Framework** | None | The design has precise spacing, colors, and layout. Bootstrap/Tailwind would fight the design, not help it. |
| **JavaScript** | None | Zero JS dependencies. Blazor handles all rendering server-side. |
| **Testing** | Optional (bUnit + xUnit) | Recommended for SVG coordinate math and JSON deserialization. Not critical for MVP. |
| **External NuGet Packages** | Zero | Everything ships with the .NET 8 SDK. |

## Security Considerations

### Threat Model

This application has a minimal attack surface by design:

| Aspect | Status | Notes |
|---|---|---|
| **Authentication** | None | Local-only access. No user accounts. |
| **Authorization** | None | Single implicit user (whoever runs the app). |
| **Data Input** | `data.json` only | No user-submitted forms, no file uploads, no query parameters. |
| **Network Exposure** | Localhost only | Kestrel binds to 127.0.0.1. Not reachable from other machines. |
| **Sensitive Data** | None | Dashboard shows project status, not PII or credentials. |
| **HTTPS** | Dev cert only | Kestrel dev certificate is sufficient for localhost. |
| **CORS** | Not applicable | No cross-origin requests (no API, no AJAX). |
| **XSS** | Mitigated by Blazor | Blazor encodes all rendered output by default. `data.json` content is rendered through Blazor's encoding pipeline, not raw HTML injection. |
| **Secrets** | None | No API keys, no connection strings, no tokens. |

### Validation

- `data.json` is validated at deserialization time via `required` record properties.
- Invalid JSON fails fast with a descriptive `JsonException`.
- Missing fields throw at startup, not at render time.
- Date strings validated when parsed via `DateOnly.Parse` with try/catch fallback.

### Hardening (If Ever Network-Exposed)

If the dashboard is ever shared on a network:
1. Bind explicitly: `builder.WebHost.UseUrls("http://localhost:5000")`
2. Add `X-Frame-Options: DENY` header

## Scaling Strategy

### Current Scale

| Metric | Value |
|---|---|
| Concurrent users | 1 |
| Data size | < 10 KB |
| Page count | 1 |
| DOM elements | ~50-100 |
| Render time | < 100ms |

### Scaling Is Not Required

This is a local single-user tool. There are no scaling concerns. The architecture explicitly avoids:
- Load balancers
- Caching layers
- Database connection pools
- CDN distribution
- Horizontal scaling

### Data Scaling

If `data.json` grows to support more months or categories:
- **6 months × 4 categories = 24 cells:** No performance impact. CSS Grid handles this natively.
- **10+ milestones:** SVG swim lanes stack vertically. Increase `SvgHeight` in `TimelineConfig`. Timeline area CSS height may need adjustment.
- **100+ items per cell:** Cell `overflow:hidden` clips excess. Consider truncation or font size reduction in CSS.

### Future Extensibility (Not In Scope, But Designed For)

The data-driven architecture supports future additions without structural changes:
- Add months → `months` array in JSON controls column count
- Add categories → `categories` array controls row count
- Add milestones → `milestones` array controls swim lane count
- Change colors → `color` field per milestone, `cssClass` per category

## Risks & Mitigations

| # | Risk | Likelihood | Impact | Mitigation |
|---|------|-----------|--------|------------|
| 1 | **JSON schema drift** — `data.json` structure changes break deserialization | Medium | High (dashboard crashes) | Use `required` keyword on all record properties. Add defensive null-coalescing in components. Fail fast with descriptive error messages at startup. |
| 2 | **SVG coordinate misalignment** — Timeline markers appear at wrong positions | Medium | Medium (visual defect) | Isolate date-to-pixel math in `DashboardDataService.CalculateXPosition`. Unit test with known date/position pairs from the reference design. |
| 3 | **CSS port drift from design** — Subtle visual differences between reference HTML and Blazor render | Low | Medium (not pixel-perfect) | Copy CSS verbatim from `OriginalDesignConcept.html`. Use a global CSS file (not scoped) to avoid `::deep` complications. Side-by-side screenshot comparison during development. |
| 4 | **Browser rendering inconsistency** — Different browsers render SVG/CSS differently | Low | Low (minor visual diff) | Standardize on Chromium (Edge or Chrome). Document browser choice. Both use identical rendering engine. |
| 5 | **Blazor scoped CSS `::deep` issues** — Styles don't cascade to child components | Medium | Medium (broken layout) | **Mitigated by design:** Architecture uses a global `dashboard.css` file, not scoped CSS. This eliminates `::deep` issues entirely. |
| 6 | **`data.json` file locked by editor** — IOException when dashboard tries to read while user is editing | Low | Low (refresh fails once) | Wrap `File.ReadAllTextAsync` in try/catch. Display "Error loading data — please retry" message. File locks are transient. |
| 7 | **Missing Segoe UI font on non-Windows OS** — Layout shifts if fallback font (Arial) has different metrics | Low | Low | Target environment is Windows. Font stack includes Arial fallback. Not a concern unless ported to macOS/Linux. |
